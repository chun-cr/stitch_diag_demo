# Session 002 - 报告页返回首页未切换修复

## 做了什么
- 排查报告页返回按钮点击后 GoRouter 打印跳转到 `/home` 但界面未切回首页的问题。
- 将首页底部导航中的“报告”入口改为通过 GoRouter 打开 `/report`，不再复用 `MainShell` 内部 `_currentIndex` 切出报告页。
- 新增 Widget Test，覆盖“首页 -> 报告 -> 返回首页”的回归链路。

## 变更前快照
- 报告页返回按钮执行 `context.go(AppRoutes.home)`，控制台会打印 `[GoRouter] going to /home`。
- `/home` 实际渲染的是 `MainShell`，但 `MainShell` 通过本地 `_currentIndex` 和 `IndexedStack` 决定显示哪个页面。
- `MainShell` 内部把 `ReportPage` 当作第 3 个 tab 页面，因此界面可能显示报告页，但当前路由仍然是 `/home`。

## 涉及文件
- `lib/features/home/presentation/pages/home_page.dart`
- `lib/features/report/presentation/pages/report_page.dart`
- `test/navigation/report_back_navigation_test.dart`

## 问题与决策
- 根因：路由状态与 `MainShell` 本地 tab 状态存在双重来源，报告页既可作为独立路由打开，也可在 `/home` 内部通过 `IndexedStack` 显示。
- 决策：不做大规模路由重构，先把报告入口改为路由驱动，避免“界面是报告页但路由仍是 `/home`”的错位状态。
- 测试策略：补一个聚焦 home -> report -> back 的回归测试，确认返回后可见页面真正切回首页。

## 验证结果
- `flutter test test/navigation/report_back_navigation_test.dart`：通过。
- `lsp_diagnostics`（`home_page.dart`、`report_back_navigation_test.dart`）：无诊断问题。
- `flutter analyze lib/features/home/presentation/pages/home_page.dart lib/features/report/presentation/pages/report_page.dart lib/core/router/app_router.dart test/navigation/report_back_navigation_test.dart`：存在既有 warning/info，本次改动未新增 analyzer 错误。

## 下次计划
- 如后续继续统一底部导航与路由，可考虑把 `MainShell` 进一步改成完全由路由驱动的壳层结构。

## 遗留风险
- 当前仅修复了报告入口的路由错位；若后续其它底部 tab 也同时存在“壳层内切页”和“独立路由页”双轨实现，仍需统一治理。
