## 七、交接建议

后续接手时，建议遵守以下规则：

1. 新增用户可见文案必须进入 ARB，不要回退到硬编码
2. 新增语言先补骨架，再补高频页面，再做全量覆盖
3. 不要把显示文案直接当业务 key 使用
4. 每次多语言变更后至少执行：
   - `flutter gen-l10n`
   - `flutter analyze`
   - 关键 smoke tests
5. 涉及 UI 变更时优先补 `maxLines / overflow / Flexible`，避免先做大改版
