# Session 021 - 面诊自动开始扫描

## 做了什么
- 排查面部扫描当前启动链路，确认现状为“人脸检测到且居中 → 用户点击开始面部扫描 → 才进入 2 秒 hold”。
- 收敛最小方案：保留现有 2 秒 hold、丢脸/偏移即取消、面诊完成后跳转舌诊的逻辑，只把启动方式改成自动开始，并移除不再需要的开始按钮与对应文案。
- 完成第一轮实现：listener 在更新 `_hasFaceDetected / _faceDirection` 后，当满足 auto-start 条件且当前未扫描时直接触发现有 `_startScan()`；底部手动开始按钮已移除，只保留“跳过此步骤”。
- 根据用户澄清继续补第二轮修正：恢复面诊底部卡中的状态提示承载区，新增 `_BottomStatusPrompt`，让页面在自动开始扫描的同时，底部仍像舌诊/掌诊一样保留明显状态提示。
- 补充纯逻辑测试 `shouldAutoStartFaceScan(...)`，并清理源 ARB 中已失效的 `scanFaceStartButton`。

## 变更前快照
- `face_scan_page.dart` 通过 `_startScan()` 启动 hold，当前唯一入口是底部按钮。
- face 页已经具备 `isFaceHoldEligible(...)` 和“中途失去 readiness 就取消 hold”的逻辑。
- `scanFaceStartButton` 当前只在 face 页使用，且源 ARB 里仅见于 `app_ja.arb` / `app_ko.arb`。

## 问题与决策
- 决策：自动开始沿用现有 `_startScan()`，不重写 hold 状态机。
- 决策：取消底部“开始面部扫描”按钮，但保留“跳过此步骤”。
- 决策：如果 `scanFaceStartButton` 在源 ARB 中彻底失去引用，则直接删除。

## 验证结果
- `flutter test test/features/scan/face_scan_hold_test.dart`：通过。
- `flutter analyze lib/features/scan/presentation/pages/face_scan_page.dart test/features/scan/face_scan_hold_test.dart`：通过。
- `lsp_diagnostics`：`face_scan_page.dart`、`face_scan_hold_test.dart` 无错误。
- `flutter gen-l10n`：已执行；仓库里显示 `ja/ko` 仍有其他历史未翻译文案提示，但本次删除未引入新的生成失败。
- Oracle 首轮复核：确认手动 CTA 已移除、listener 只会在 ready 且 idle 时自动触发 hold、readiness 丢失的取消逻辑保持不变、`scanFaceStartButton` 清理安全。
- Oracle 二轮复核：确认 face 页现在已恢复底部状态提示区，同时保留自动开始扫描，不存在手动开始 CTA 回流，符合“像舌头扫描、手掌扫描一样保留提示状态”的澄清要求。
