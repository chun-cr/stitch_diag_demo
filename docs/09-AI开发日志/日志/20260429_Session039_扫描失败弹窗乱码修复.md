# Session 039 - 扫描失败弹窗乱码修复

## 做了什么
- 复核 Android 真机上传失败弹窗的文案来源，确认 `scan_debug_error_dialog.dart` 只负责展示传入的 `title` 与错误描述，乱码是否出现取决于调用侧标题字符串和异常文本来源。
- 确认当前用户可见乱码出现在 `lib/features/scan/presentation/pages/tongue_scan_page.dart` 的失败弹窗标题：`一条已损坏的上传失败标题字符串`。
- 已将扫描失败弹窗标题最小收口到 `l10n`，同步覆盖 face / tongue 两个调用点，避免继续在页面里保留硬编码中文标题。

## 变更前快照
- `face_scan_page.dart` 失败弹窗标题为硬编码中文 `人脸上传失败`。
- `tongue_scan_page.dart` 失败弹窗标题为已损坏的乱码字符串（已清理）。
- `scan_debug_error_dialog.dart` 的 `content` 直接展示 `ScanUploadException.debugDescription` / `ScanCaptureException.debugDescription`，因此标题修复后若正文仍乱码，需要另查运行时异常来源。

## 问题与决策
- 决策：不只替换乱码字面量，而是把 face / tongue 失败弹窗标题一起收进 `l10n`，避免继续扩大硬编码用户可见文案。
- 决策：本次只处理标题，不改 `scan_debug_error_dialog.dart` 的正文拼装逻辑；如果真机正文仍乱码，后续单独排查 `error.debugDescription` 的运行时来源。

## 下次计划
- 若真机上弹窗正文仍有乱码，下一步单独排查 `ScanUploadException.debugDescription` / `ScanCaptureException.debugDescription` 中的后端 message 或原生 `PlatformException.message/details`。
