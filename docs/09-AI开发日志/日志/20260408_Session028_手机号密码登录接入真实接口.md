# Session 028 - 手机号密码登录接入真实接口

## 做了什么
- 将登录页从 preview 模拟提交改为通过真实接口 `/api/v1/saas/mobile/auth/login/password` 发起手机号密码登录。
- 将账号创建页接入真实接口 `/api/v1/saas/mobile/auth/register/password`，成功后保存 token session 并进入资料完善页。
- 新增 token 会话模型与本地持久化：保存 `accessToken`、`refreshToken`、`tokenType`、`expiresIn`、`scope`。
- 将 `AuthInterceptor` 接入本地 token 读取，为非 auth 接口自动附加 `Authorization` 请求头。
- 在应用启动时根据本地 token 恢复预览登录态；退出登录时同步清除本地 token。
- 增补登录页失败回归：服务端返回 `message` 时，页面显示服务端错误信息且按钮恢复 idle。
- 增补注册页成功/失败回归：成功时进入 `CompleteProfilePage`，失败时显示服务端返回的 `message`。
- 根据 Oracle 复核，将 token 解析改为严格模式，避免 access token 缺失时误判为登录成功。
- 修复登录页第三方登录按钮在 Windows 端的轻微横向溢出。

## 变更前快照
- `login_page.dart` 提交后仅做本地延时与页面退场，然后直接 `setPreviewAuthenticated(true)` 进入首页。
- `register_page.dart` 的创建账号按钮仅做本地延时后跳转资料完善页，没有真实注册请求。
- `AuthRemoteSource.login()` 已指向真实登录 URL，但返回体仍错误假设为用户信息模型。
- `AuthRemoteSource.register()` 也仍错误假设为用户信息模型，而不是 token session。
- `AuthInterceptor` 为空实现，不会携带 token。
- `main.dart` 未初始化 injector，也不会根据本地登录会话恢复登录态。
- `profile_page.dart` 退出登录只清 preview 状态，不清 token。

## 问题与决策
- 决策：沿用现有 `AuthRequest(countryCode, phoneNumber, password)`，直接对接真实密码登录接口，不引入新的登录表单结构。
- 决策：沿用同一 `AuthRequest(countryCode, phoneNumber, password)`，将账号创建页也直接对接真实密码注册接口，不引入验证码流。
- 决策：登录成功返回体按 `{ data: { accessToken, refreshToken, tokenType, expiresIn, scope } }` 解析为独立 session entity，而不是继续错误映射成用户信息。
- 决策：注册成功返回体与登录保持同构，也解析为独立 session entity；注册成功后保存 session，再进入资料完善页。
- 决策：token 持久化先使用项目现有 `SharedPreferences`，不额外引入 secure storage 依赖。
- 决策：应用启动时以“是否存在 accessToken”作为 preview 登录态恢复依据；保持当前 preview router 模型，不额外引入完整 auth state machine。
- 决策：`AuthInterceptor` 仅为非 `/api/v1/saas/mobile/auth/` 请求附加 `Authorization` 头，避免登录/注册接口带无意义旧 token。
- 决策：登录失败时优先显示服务端返回的 `message`，否则回退到本地化通用文案 `authLoginFailed`。
- 决策：注册失败时优先显示服务端返回的 `message`，否则回退到本地化通用文案 `registerCreateFailed`。
- 决策：根据 Oracle 复核，移除了登录前对空手机号/空密码自动补默认值的逻辑，避免绕过真实校验。
- 决策：根据 Oracle 复核，`AuthSessionModel` 不再对 token 字段静默给空串/0 默认值，缺字段直接抛错，避免“假成功”写入本地 session。
- 决策：登录页成功路径测试改为 provider override + fake repository，避免 widget test 直接命中真实网络。
- 决策：注册页成功跳资料完善页和失败消息展示同样改为 provider override + fake repository，避免 widget test 直接命中真实网络。
