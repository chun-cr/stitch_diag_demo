# Session 004 - 首页历史记录页接入与趋势页实现

## 做了什么
- 在首页“最近一次辨证摘要”卡片标题行增加“查看全部 >”入口，并让整张白卡可点击跳转到历史记录页。
- 将首页底部三项评分条改成扁平、绿色渐变、圆角进度条样式。
- 新增 `HistoryReportPage`，实现健康走势折线图、风险指数折线图和双列历史报告卡片网格。
- 接入 `fl_chart` 与 `cached_network_image` 依赖，并将 `AppRoutes.history` 从占位页接到真实页面。
- 对齐首页最近一次记录与历史页 mock 数据来源，避免首页日期/体质与历史记录首条不一致。

## 变更前快照
- 首页最近一次检测记录卡片没有“查看全部”入口，也没有点击跳转历史页的交互。
- `AppRoutes.history` 仍指向占位 `_Placeholder(label: '历史记录')`。
- 项目当前未引入 `fl_chart` 与 `cached_network_image`。

## 涉及文件
- `lib/features/home/presentation/pages/home_page.dart`
- `lib/core/router/app_router.dart`
- `lib/features/history/presentation/pages/history_page.dart`（新增）
- `pubspec.yaml`

## 问题与决策
- 决策：保持现有 GoRouter 主体结构不变，同时实现首页摘要卡和“查看全部”的页面跳转。
- 决策：历史页数据先采用页面内静态 mock 结构承载，避免本次任务扩散到状态管理与后端模型。
- 决策：首页摘要卡三项评分条同步改成扁平、绿色渐变圆角进度条，与报告页语言统一。

## 验证结果
- `flutter pub get`：成功，新增依赖已安装。
- `lsp_diagnostics`（`home_page.dart`、`app_router.dart`、`history_page.dart`）：无诊断问题。
- `flutter analyze lib/features/home/presentation/pages/home_page.dart lib/core/router/app_router.dart lib/features/history/presentation/pages/history_page.dart`：无错误；剩余为既有 `home_page.dart` info/warning（未引用私有组件、unnecessary_underscores）。
- Oracle 复核结论：功能已满足演示级要求，主要风险在于当前仍是 mock 数据驱动；已通过共享同一份示例记录减轻首页与历史页数据不一致问题。

## 下次计划
- 若用户继续扩展历史数据能力，可再接入真实记录源和解锁逻辑。

## 遗留风险
- 当前历史页仍使用页面内 mock 数据，尚未接入真实历史记录与解锁逻辑。
- 历史卡片图片依赖外部网络图片地址，弱网/离线场景下会回退为占位图。
