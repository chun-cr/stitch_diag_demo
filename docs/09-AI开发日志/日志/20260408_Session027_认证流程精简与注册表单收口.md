# Session 027 - 认证流程精简与注册表单收口

## 做了什么
- 登录页改为纯手机号登录：删除邮箱模式切换与邮箱相关输入逻辑，保留国家区号选择器、微信登录与 Apple 登录。
- 结合新的漏斗设计，将注册流程进一步拆分：`register_page.dart` 收缩为纯账号创建页，只保留手机号、密码、确认密码、协议与创建账号按钮。
- 新增独立的 `CompleteProfilePage` 作为资料完善页，承接头像、昵称、性别，并提供“跳过”入口；头像上传区改为静态圆形占位 + 白色相机小圆按钮，性别改为轻量描边药丸标签。
- 注册页移除第三方登录入口与两步 PageView/步骤条；注册成功后跳转到新的 `/complete-profile` 路由，完成或跳过后进入 `home`。
- 注册页移除呼吸/脉冲动画，只保留静态视觉与现有旋转装饰；密码强度与确认密码校验保持不变。
- 删除不再使用的邮箱登录相关 ARB key，并补充 focused widget tests 与日志验证结果。
- 根据 Oracle 复核，额外移除了登录提交前对空手机号/空密码自动补默认值的逻辑，避免绕过新增校验。

## 变更前快照
- `login_page.dart` 仍保留手机号/邮箱双模式切换，默认是邮箱模式。
- 登录页手机号模式使用国家区号前缀菜单，但手机号格式尚未做显式校验。
- `register_page.dart` 第一步仍使用“姓名 + 手机号/邮箱 + 性别（选填）”。
- 注册页性别选择状态保存在 `_GenderSelector` 子组件内部，父表单无法直接校验其必填性。
- 注册页存在 `_pulseController` 驱动的呼吸抖动视觉。

## 问题与决策
- 决策：登录页移除邮箱登录逻辑，仅保留手机号登录输入、微信登录、Apple 登录。
- 决策：登录页手机号格式采用统一规则：仅允许 6-15 位纯数字，本地号段与区号前缀分离校验。
- 决策：注册页不再承载资料补全和第三方登录，只负责账号创建；资料补全拆到独立 `CompleteProfilePage`。
- 决策：注册页采用手机号 + 密码方案，验证码流程按用户选择留待下一轮单独实现。
- 决策：注册成功后先进入 `/complete-profile`，该页允许 `Skip`；完成或跳过时再调用 `setPreviewAuthenticated(true)` 进入首页，以避免现在就引入新的 onboarding 状态机。
- 决策：昵称与性别从注册页移出后，`CompleteProfilePage` 自己持有表单状态，并在点击“开启体验”时要求昵称和性别有效；点击“跳过”则直接放行。
- 决策：密码仍按现有强度和确认密码规则校验，不降低门槛。
- 决策：注册页呼吸抖动属于可删减视觉噪音，已移除 `_pulseController` 及其相关 AnimatedBuilder，只保留静态外环与旋转背景装饰。
- 决策：为避免遗留死词条，删除了不再使用的 `authEmailLabel`、`authEmailOrPhoneLabel`、`authEmailOrPhoneHint`、`authSwitchToEmailLogin`、`authSwitchToPhoneLogin`、`registerNeedBasicInfo`，以及旧注册两步流相关的 `registerStepBasicInfo`、`registerStepSetPassword`、`registerSetPasswordTitle`、`registerSetPasswordSubtitle`、`registerNextStep`、`registerComplete`、`registerThirdParty`、`registerWechat`。
- 决策：登录页不再允许通过默认补值绕过手机号/密码校验；preview/demo 值从正式提交链路中完全移除。
