# Session 001 - 登录页预览跳转修复

## 做了什么
- 为 `GoRouter` 增加开发预览态登录守卫，未登录时统一收口到 `/login`。
- 调整登录页预览逻辑：默认填充演示账号密码，点击登录按钮即可进入首页。
- 新增 Widget Test，覆盖“应用启动进入登录页、点击登录进入首页”的链路。

## 变更前快照
- Web/Chrome 端命中 `/home` 时会直接进入首页，未经过登录页。
- 登录页按钮依赖手输账号密码，不适合当前开发预览场景。
- 原有 `widget_test.dart` 仍是默认计数器示例，无法覆盖当前登录流问题。

## 涉及文件
- `lib/core/router/app_router.dart`
- `lib/features/auth/presentation/pages/login_page.dart`
- `test/widget_test.dart`

## 问题与决策
- 根因：Web 端缺少未登录路由守卫，浏览器地址栏若落在 `/home` 会直接渲染首页。
- 决策：不引入新依赖，不接真实鉴权，先用内存态预览登录状态满足当前“仅预览页面”的需求。
- 测试补充：首页存在初始化延时动画，测试中额外补泵帧以消费定时器，避免误报失败。

## 验证结果
- `flutter test`：通过。
- `flutter build web`：通过。
- `lsp_diagnostics`（本次改动文件）：无诊断问题。
- `flutter analyze`：项目仍有既有 warning/info，本次改动未新增 analyzer 问题。

## 下次计划
- 若后续接入真实账号体系，可把当前预览登录态替换为持久化鉴权状态，并补充退出登录链路。

## 遗留风险
- 当前登录态仅为内存态，刷新页面后会回到登录页；这符合当前“预览登录页后点击进入首页”的目标，但不等同于正式登录实现。
