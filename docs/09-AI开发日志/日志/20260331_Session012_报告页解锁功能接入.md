# Session 012 - 报告页解锁功能接入

## 做了什么
- 在 `report_page.dart` 增加报告解锁状态与统一解锁回调，准备接入后续支付逻辑。
- 为报告页新增锁定包裹组件与遮罩组件，并按要求对体质、调理、建议三个 Tab 的指定模块做模糊锁定。
- 对总览 Tab 的模块入口增加点击拦截：未解锁时弹出底部解锁弹窗，已解锁时正常切换 Tab。
- 补齐报告解锁相关多语言文案，确保 `zh / en / ja / ko` 一致可生成。

## 变更前快照
- 报告页仅展示静态内容，没有统一的“解锁完整报告”状态控制。
- 总览模块入口点击逻辑为空，体质/调理/建议 Tab 的内容也没有分级锁定。
- 解锁文案只存在于 `app_zh.arb`，其余语言未补齐。

## 涉及文件
- `lib/features/report/presentation/pages/report_page.dart`
- `lib/l10n/app_zh.arb`
- `lib/l10n/app_en.arb`
- `lib/l10n/app_ja.arb`
- `lib/l10n/app_ko.arb`

## 问题与决策
- 决策：不改现有模块内部实现，只在指定方法调用处外包 `_Lockable`，满足最小侵入约束。
- 决策：总览入口不使用 overlay，而是沿点击链路拦截并复用统一底部解锁弹窗。
- 决策：锁定标题使用独立多语言 key，遮罩副标题与按钮优先复用已有通用解锁文案。

## 验证结果
- `flutter gen-l10n`：通过；控制台提示 `en / ja / ko` 仍有 9 条未翻译消息，为项目现存提示，不影响本次新增 key 生成。
- `flutter analyze lib/features/report/presentation/pages/report_page.dart`：通过。
- `lsp_diagnostics`：`lib/features/report/presentation/pages/report_page.dart` 无错误。

## 下次计划
- 支付链路明确后，将 `_handleUnlock` 中的本地模拟逻辑替换为真实支付成功回调。

## 遗留风险
- 若后续支付接入需要区分模块级购买，当前统一 `_isUnlocked` 状态需要再抽象。

### 本次追补（Apple IAP 接入）
- 新增 `lib/features/report/application/report_unlock_service.dart`，使用官方 `in_app_purchase` 插件承接报告解锁、购买流监听、本地已购缓存与恢复购买。
- 报告页将原 `_handleUnlock` 本地 mock 解锁替换为统一 paywall 入口，所有锁定入口均改为进入 Apple IAP 弹层，不再直接改本地布尔值。
- 解锁弹层新增真实商品价格展示、购买中/恢复中状态、恢复购买入口，以及 App Store 不可用/商品不存在/购买失败等提示文案。
- 在 `pubspec.yaml` 增加 `in_app_purchase` 官方依赖；商品 ID 通过 `REPORT_UNLOCK_PRODUCT_ID` dart-define 可覆盖，默认值为 `report_unlock_full`。
