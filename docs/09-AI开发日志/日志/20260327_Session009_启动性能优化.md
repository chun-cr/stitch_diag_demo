# Session 009 - 启动性能优化

## 做了什么
- 按优先级完成启动性能优化：先移除 Android 冷启动期的扫描模型初始化，再降低登录页首屏持续动画负担，最后减少首页重页面预构建。
- Android 侧将 `FaceLandmarkerHelper` / `GestureRecognizerHelper` 改为首次真正使用时才创建，避免 App 启动即加载 MediaPipe 模型。
- 登录页取消持续旋转背景与旋转八卦环，改为静态背景 + 轻量呼吸/淡入效果，减少首屏重绘。
- 首页 `MainShell` 移除对 `ReportPage` 等非当前页的预挂载，改为只构建当前显示页。

## 变更前快照
- Android 在 App 启动阶段即创建面部与手势识别 helper，可能提前加载 MediaPipe 模型。
- 登录页首屏包含持续旋转与多层 `CustomPaint` 动画，首帧和持续重绘成本偏高。
- 首页 `MainShell` 预挂载了并不会立即使用的重页面，放大登录后首进首页卡顿。

## 涉及文件
- `android/app/src/main/kotlin/com/example/stitch_diag_demo/FaceScanChannel.kt`
- `lib/features/auth/presentation/pages/login_page.dart`
- `lib/features/home/presentation/pages/home_page.dart`

## 问题与决策
- 决策：不动扫描算法与路由结构，优先消除明显的冷启动非必要工作。
- 决策：保留页面视觉风格，但削减持续动画与首屏多余重绘。
- 决策：首页保留现有导航交互，仅移除不必要的预构建页面实例。
- Oracle 复核结论：优化顺序保持 `原生懒初始化 -> 登录页减载 -> 首页减载` 是合理的；channel/platform view 注册保留启动时执行，仅延后 helper 实例创建。

## 验证结果
- `lsp_diagnostics`：`login_page.dart`、`home_page.dart` 无错误；Kotlin LSP 当前环境未安装，改用 Android 编译校验。
- `flutter test`：全部通过。
- `android\\gradlew.bat :app:compileDebugKotlin`：通过。
- `flutter analyze lib/features/auth/presentation/pages/login_page.dart lib/features/home/presentation/pages/home_page.dart`：无新增错误；剩余为项目既有 `unused_element` / `unnecessary_underscores` 提示，不属于本次启动优化新增问题。
