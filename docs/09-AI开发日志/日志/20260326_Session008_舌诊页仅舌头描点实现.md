# Session 008 - 舌诊页仅舌头描点实现

## 做了什么
- 将舌诊页 overlay 从“直接绘制整脸 landmarks”改为“仅基于嘴部点推导舌头区域描边”。
- 仅在识别到伸舌时显示舌头近似描点，不再显示整张脸的点云。

## 变更前快照
- Flutter 侧 `tongueLandmarks` 实际消费的是原生回传的整脸 `Face Landmarker` 点集。
- 舌诊页视觉上会出现整脸描点，不符合“只描舌头”的产品预期。

## 涉及文件
- `lib/features/scan/presentation/pages/tongue_scan_page.dart`
- `lib/features/scan/presentation/widgets/tongue_landmark_overlay.dart`

## 问题与决策
- 决策：不改模型前提下，不再尝试伪装整脸点为舌头点，改为基于 `mouthLandmarks` 与 `tongueOutScore` 推导近似舌形。
- 决策：只在 `tongueDetected = true` 时显示舌头描边，未识别到时不显示误导性点云。

## 验证结果
- 待修改完成后补充 `flutter analyze` 与 Oracle 复核结果。

## 下次计划
- 若后续接入舌头分割或舌头关键点模型，可把近似舌形替换为真实舌体轮廓。

## 遗留风险
- 当前实现是基于嘴部点的近似舌形，不是真实舌头几何点；视觉正确性高于医学精度。
