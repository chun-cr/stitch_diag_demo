# Session 005 - 舌头扫描描点逻辑修复

## 做了什么
- 修复舌头扫描事件在 Android/iOS 两端字段不一致的问题，统一补齐舌扫描点、嘴部中心点与图像尺寸。
- 修复嘴部中心点被混入嘴唇轮廓点数组，导致 Flutter 侧嘴部描线异常的问题。
- 修复 Flutter 舌扫页对镜像的处理，恢复按平台区分 overlay 映射方向，并补充桥接层单元测试。

## 变更前快照
- Android 舌扫事件仅向 Flutter 发送 `tongueDetected`、`tongueOutScore`、`mouthLandmarks`、`landmarks`，缺失 `imageWidth`、`imageHeight` 与独立 `mouthCenter`。
- iOS 舌扫事件只发送嘴部轮廓和尺寸，不发送 `landmarks`，导致 Flutter 侧舌扫描点为空。
- 原生侧把嘴部中心点直接追加到 `mouthLandmarks` 数组，Flutter 绘制轮廓时会把中心点当成边界点连线。

## 问题与决策
- 决策：不改扫描算法阈值，先修数据契约和坐标映射，避免把“描点看起来不对”误判成检测灵敏度问题。
- 决策：嘴部中心点改为独立字段 `mouthCenter`，轮廓数组只保留边界点，避免 UI 层再做猜测性过滤。
- 决策：沿用现有 MediaPipe face landmarks 作为舌扫描点来源，不额外引入新模型或第三方依赖。
