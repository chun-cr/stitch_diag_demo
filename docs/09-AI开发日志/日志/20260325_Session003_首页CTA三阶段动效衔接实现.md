# Session 003 - 首页CTA三阶段动效衔接实现

## 做了什么
- 为首页“开始全套智能检测”按钮实现三阶段动效骨架：按下微缩与触觉反馈、按钮形变为圆形加载态、绿色 reveal 全屏扩散。
- 将 `/scan` 入口改为零转场切页，并在 `ScanGuidePage` 增加首帧绿色遮罩淡出与页面内容淡入，形成首页到扫描引导页的无缝衔接。
- 根据 Oracle 复核建议，进一步消除了 `/scan` 返回时的非零反向过渡，并缩短了扫描页首帧纯绿停顿时间。

## 变更前快照
- 首页 CTA 点击后直接 `context.push(AppRoutes.scan)`，只有静态按钮样式，没有任何状态转变或过渡层。
- `/scan` 使用普通 `GoRoute(builder: ...)`，默认由 GoRouter/Flutter 负责切页。
- `ScanGuidePage` 进入时直接渲染宣纸米色页面和内容卡片，没有承接首页 CTA reveal 的视觉过渡。

## 涉及文件
- `lib/features/home/presentation/pages/home_page.dart`
- `lib/features/scan/presentation/pages/scan_guide_page.dart`
- `lib/core/router/app_router.dart`

## 问题与决策
- 决策：不引入第三方动效库，完全基于 Flutter 自带 `AnimationController`、`CustomPainter`、`FadeTransition` 与 `HapticFeedback` 实现。
- 决策：首页负责出场动画（morph + reveal），扫描引导页负责入场接管（绿色遮罩淡出 + 内容淡入），避免把复杂 reveal 直接耦合进 GoRouter 的通用页面过渡。
- 决策：`/scan` 改为零转场 `CustomTransitionPage`，减少 reveal 过程被默认路由动画打断的风险。

## 验证结果
- `lsp_diagnostics`（`home_page.dart`、`scan_guide_page.dart`、`app_router.dart`）：无诊断问题。
- `flutter analyze lib/features/home/presentation/pages/home_page.dart lib/features/scan/presentation/pages/scan_guide_page.dart lib/core/router/app_router.dart`：无新增 analyzer 错误；仍有 `home_page.dart` 既存 warning/info（未引用私有组件、unnecessary_underscores）。
- Oracle 复核结论：整体方案可用、架构无严重问题；已按建议修正 `/scan` 反向转场和首帧纯绿停顿过长的问题。

## 下次计划
- 如用户继续追求更强的“水滴落入池塘”质感，可进一步把 reveal 扩展为双向过渡（进入与返回都具备收束/展开动画）。
- 如需要更电影化的启动感，可继续把圆形 loader 从普通旋转弧线升级为更具经络感的定制路径动画。
