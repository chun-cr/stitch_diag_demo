# Session 008 - 舌头扫描点位与检测偏移修复

## 做了什么
- 将舌头扫描 overlay 从“实心舌形区域”改为 5 个稀疏关键点，并补一条轻量引导线，视觉效果向面部描点靠拢。
- 将 Android/iOS 舌头判定阈值统一并小幅放宽，恢复舌头检测灵敏度。
- 为 Flutter 侧舌头点位生成与映射逻辑补充单元测试，覆盖镜像与 aspect-fill 映射。

## 变更前快照
- 当前 `TongueLandmarkOverlay` 会基于嘴部点推导整块舌形路径并做渐变填充，视觉上像一片实心区域。
- Android/iOS 原生舌头判定阈值不一致，且 Android 阈值明显更严格。
- 用户反馈当前舌头描点不够灵敏，且点位相对实际舌头区域向右偏移。

## 问题与决策
- 决策：先取消舌头实心区域绘制，改为少量稀疏点位，避免合成舌形带来的视觉偏移被继续放大。
- 决策：统一 Android/iOS 舌头判定阈值，并做小幅放宽，优先恢复用户感知上的识别灵敏度。
- 决策：不引入新模型、不改事件契约，继续沿用现有 MediaPipe face landmarks 与 blendshape 结果。
