# Session 017 - 舌头阈值与扫描交互统一

## 做了什么
- 启动排查三项问题：舌头扫描能识别口部但无法稳定识别伸舌；手部扫描缺少明显“开始保持 2 秒”的视觉交互；面部扫描仍使用数字倒计时而不是舌头式进度反馈。
- 收敛最小方案：先通过测试锁定舌头 bridge 阈值边界，再把 palm / face 的提示统一到“气泡 + 2 秒进度”模型。
- 编写并评审 `.sisyphus/plans/scan-hold-and-tongue-fix.md`，补齐任务级 QA 和运行时检查点。
- 完成实现：将 `tongue_scan_status_bridge.dart` 的就绪阈值从 `0.26` 对齐到 `0.20`，并同步更新舌头 bridge 回归测试边界。
- 完成 face / palm 页面交互统一：face 由数字倒计时改为 tongue-like 气泡 + 2 秒进度；palm 的 hold 状态改为更明确的气泡 + 进度组合。
- 根据 Oracle 复核补齐 face hold 状态机：只有在人脸持续存在且保持居中时才继续 hold；一旦丢脸或偏移，立即取消并清零进度。

## 变更前快照
- `tongue_scan_status_bridge.dart` 当前 `readyToScan` 阈值是 `0.26`，但 native 双端 `tongueOut` 阈值是 `0.2`，存在正例带宽不一致。
- `TongueScanPage` 已有目标交互模型：状态气泡 + 2 秒进度条。
- `PalmScanPage` 虽已有气泡和进度条，但交互感知仍不强；`FaceScanPage` 仍保留数字倒计时风格。

## 问题与决策
- 决策：先修舌头 bridge 阈值，不先动原生阈值；这是跨端最小且可立即见效的修复点。
- 决策：face / palm 交互统一只改页面层，不改路由和业务链路。
- 决策：优先复用现有本地化 key，只有在确实缺文案时才新增 ARB。
