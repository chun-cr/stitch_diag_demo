# Session 023 - 登录按钮两段式动效优化

## 做了什么
- 排查登录页当前点击动效，确认现状为“按压缩放 → 立即切 loading → 等待后整页退场”。
- 第一轮曾尝试做成 `idle -> confirming -> loading -> page exit` 的轻量承接，但用户明确认为圆圈和内部切换仍然不够严肃。
- 第二轮按用户要求继续做减法：去掉确认箭头、去掉内部旋转圆圈、去掉按钮内容切换动画，只保留轻按压、按钮压暗禁用，再交给页面退场。
- 最终实现：登录按钮状态收敛为 `idle -> submitting -> page exit`，并保留现有登录页整体轻量性能方向。

## 变更前快照
- `login_page.dart` 最初使用 `_btnScaleCtrl`、`_spinCtrl`、`AnimatedSwitcher` 和 `_exitCtrl`，按钮内部会切箭头 / 圆圈 / loading 等状态。
- 这种按钮内部演出感对登录场景来说偏轻秀，不够克制。

## 涉及文件
- `lib/features/auth/presentation/pages/login_page.dart`
- `test/features/auth/presentation/pages/login_page_test.dart`

## 问题与决策
- 决策：不引入首页 CTA 那种重 reveal，继续坚持登录页“轻、稳、短”的方向。
- 决策：最终不再做按钮内部确认箭头或 spinner，只保留单一 submitting 态。
- 决策：不新增文案 key，按钮文案继续复用现有登录文案，避免无收益的多语言扩散改动。
- 决策：测试聚焦按钮状态简化本身，不把路由跳转细节耦合进这个动效测试；导航链路继续由现有导航回归测试承担。

- Oracle 只读复核：确认当前实现已收敛成更严肃、克制的 `idle -> submitting -> page exit`；补充提示是测试主要锁“去掉 playful phase”，而不是逐帧锁视觉细节，这一点当前是可接受的。

## 下次计划
- 如需继续增强登录体验，优先考虑再微调压暗程度和按压缩放幅度，而不是重新引入按钮内部小动效。
- 若后续接入真实鉴权，可把当前 submitting 态复用到真实登录请求链路。

## 遗留风险
- 当前优化仍是页面内局部交互，没有抽成可复用 CTA 组件。
