# Session 015 - 扫描页提示样式统一

## 做了什么
- 启动排查 face / tongue / palm 三个扫描页的提示样式差异，确认舌头扫描页的小气泡提示是本次统一目标。
- 收敛最小改动方案：face 页将全屏倒计时改为框下小气泡；palm 页移除底部大反馈卡，仅保留框下小气泡作为主提示。
- 编写并评审 `.sisyphus/plans/scan-prompt-ui-unification.md`，补齐运行时 QA 场景，确保这次改动只调整提示表现，不改扫描逻辑。
- 完成实现：`FaceScanPage` 删除全屏倒计时蒙层，改为框下 `_CountdownPill`；`PalmScanPage` 删除底部 `_LiveFeedbackCard`，并收紧底部按钮文案，避免继续充当大提示框。
- 补充气泡文本溢出保护，为 face / palm 的状态气泡、方向气泡和倒计时气泡增加 `maxLines + ellipsis`，降低多语言挤压风险。

## 变更前快照
- `FaceScanPage` 在 `_isScanning && _countdown > 0` 时会显示全屏暗色蒙层和大号数字倒计时，视觉权重明显高于 tongue / palm 页。
- `TongueScanPage` 已经使用框下 `_StatusPill / _TongueDirectionPill` 作为轻量提示，是当前最接近目标的样式基线。
- `PalmScanPage` 同时存在框下状态气泡和底部 `_LiveFeedbackCard`，用户会收到重复反馈。

## 涉及文件
- `lib/features/scan/presentation/pages/face_scan_page.dart`
- `lib/features/scan/presentation/pages/palm_scan_page.dart`
- `.sisyphus/plans/scan-prompt-ui-unification.md`

## 问题与决策
- 决策：不改 face / palm 的扫描状态机、读条和跳转逻辑，只替换提示承载形式。
- 决策：tongue 页作为样式基准，优先复用现有小气泡语义与布局，而不是新建大块反馈组件。
- 决策：优先复用现有本地化 key，例如 `scanKeepStill`、`scanScanning`、`scanPalmHoldButton`，非必要不新增 ARB 文案。
