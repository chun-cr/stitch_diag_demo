# Session 022 - 扫描相机切换与阈值调整

## 做了什么
- 排查三项需求：面诊扫描中底部提示改成舌诊式灰色“扫描中...”、舌头扫描张嘴 fallback 阈值从 0.08 调到 0.06、在扫描页顶部右上角加入前后摄切换按钮。
- 确认舌头 fallback 阈值当前位于 Android `TongueDetectionUtils.kt` 与 iOS `TongueDetectionEvaluator.swift`，不是 Dart bridge。
- 确认相机切换链路存在半成品：`FaceScanStatusBridge` 已有 `toggleCamera()`，`AppDelegate.swift` 已暴露 `face/toggleCamera`，但舌诊/掌诊 bridge 与原生 CameraManager 仍未真正补齐。
- 完成实现：面诊扫描中底部状态改为灰色 `scanScanning` 文案；掌诊 bridge 补齐 `toggleCamera()`；face / tongue / palm 顶栏右上角统一接入前后摄切换按钮；Android/iOS 原生相机切换链路通过共享 `face/channel` 打通。
- 原生舌头 fallback 阈值已从 `0.08` 调整为 `0.06`。

## 变更前快照
- `face_scan_page.dart` 底部状态框扫描时仍是高亮绿态，不是舌诊那种灰色“扫描中”。
- Android / iOS 原生 tongue fallback 仍使用 `mouthOpenRatio >= 0.08`。
- 舌诊页右上角已有半成品翻转按钮，但 bridge 未实现；face / palm 右上角仍是留白。

## 问题与决策
- 决策：面诊底部状态框在扫描时改成灰色“扫描中”，但不改框下 hold feedback。
- 决策：阈值调整只改原生 fallback `0.08 -> 0.06`，不直接扩大到其他 tongue score 语义。
- 决策：相机翻转优先补齐现有 `face/channel` 路径，不另起一套新的 platform channel。
